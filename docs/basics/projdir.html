The project directory

Any project resides in its own project-directory with a predefined (auto-generated) minimal-structure of 4 specific files plus these 4 folders:

  • default-build (generated output)
  • pages (all authored content source pages: likely-mostly-unstyled/unthemed/inner-body-only HTML files)
  • static (raw files to batch-copy unprocessed straight into default-build)
  • tmpl (templates and snippets)

Get started

Run HaXtatic specifying "the project-directory to process" and any missing parts from the above essentials will be (re)created in-place.

Example, to create a new site project named mydemosite.foo (inside the existing ~/dev/ directory in this case)

  • just run from inside the haXtatic code-base's src directory:
    stack runghc ./@HaxMain.hs ~/dev/mydemosite.foo
  • or with a compiled executable binary (eg. for extra speed if so inclined), of course
    my-haxtatic-exe ~/dev/mydemosite.foo works equally
and the following is generated:

Now just proceed creating/modifying contents/templates/settings/etc. within this directory structure and then simply run the same command-line again to refresh default-build with any changes made since the last run.

Of note: haXtatic never deletes any files whatsoever, and never writes any files outside the designated build folder(s) — except to re-create the few default folders/files described here, if missing.

4 standard folders

Directory default-build:

Contains the complete generated site after a successful run. (Technically, contains all outputs from all previous runs ever since the directory was created or last cleaned up manually.)

Directory static:

All (changed or new) files and folders (if any) will just be batch-copied over straight into default-build (with no processing) as the first step.

Directory pages:

All (changed or new) files and folders in here will then be processed by applying a template and the resulting corresponding output file goes into default-build; if pages has no files at all, only then a default index.html will be created once (to get new projects started with).

Directory tmpl:

Contains the auto-generated default templates and snippets plus any custom ones, used during processing of the content source files from pages.

4 standard files

Always created if missing:

  • default.haxproj — initially near-empty file (and may well be completely empty) that will over time contain all sorts of custom project-specific definitions used in the project's content source pages, snippets or templates.
  • tmpl/default.haxtmpl.html — contains the complete outer HTML scaffolding markup that every generated HTML page will embed its own unique (processed) contents into. If empty, all generated output HTML pages will be empty. Needs at least the {P|:content:|} haxTag for output pages to contain their source pages' processed contents.
  • tmpl/_hax_blokindex.haxsnip.html — unused in the initial minimum default structure and can be empty; used when auto-generating index/archive/article-listing pages for bloks.

Only created in new projects:

  • pages/index.html — only created whenever the pages directory is totally devoid of any files, as a handy example content source page to get started with.
