hax.snippet

Outputs the content of the named snippet with the specified variable substitutions.

In a nutshell

  1. Declaration of an |X|hax.snippet:myTag: .. directive in a *.haxproj file,
    • further configuring: vars, flags, content
  2. Usage anywhere: {X|myTag: vars = [..], flags = [..], content = ".." |}

Stage:

Early — this X-renderer does not require a page context for rendering: therefore any hax.snippet-invoking X-tags encountered during pre-templating (at start-up time) will be processed eagerly (for overall-reduced per-page processing load).
