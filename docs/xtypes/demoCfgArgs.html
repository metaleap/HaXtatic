Early — this X-renderer does not require a page context for rendering: therefore any hax.demoCfgArgs-invoking X-tags encountered during pre-templating (at start-up time) will be processed eagerly (for reduced per-page processing load).

Purpose

This "verging on useless" X-renderer exists solely to serve as a code sample to start from when wanting to develop custom X-renderers within the HaXtatic code-base in Haskell.
