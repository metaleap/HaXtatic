hax.htmlImage

Outputs <img .. /> or <a ..><img .. /></a>.

In a nutshell

  1. Declaration of an |X|hax.htmlImage:myTag:some/rel/base/path: .. directive in a *.haxproj file,
    • further configuring: attrLink, attrImg
  2. Usage anywhere: {X|myTag:rel-to-base/path/no/whitespace and optionally some description text |}

Stage:

Early — this X-renderer does not require a page context for rendering: therefore any hax.htmlImage-invoking X-tags encountered during pre-templating (at start-up time) will be processed eagerly (for overall-reduced per-page processing load).

Purpose

In a similar spirit to hax.miniTag and hax.htmlLink, this one is for saving precious keypresses with <img ..>-rich HTML contents requiring repetetive use of quickly-becoming-too-verbose
