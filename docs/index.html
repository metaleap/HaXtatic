Overview

These are the user docs on HaXtatic:

  • neither the first, nor the last .. neither the most powerful, nor the most minimalist .. simply just, wait for it ..
  • yet-another static site generator! You heard that right.
  • Step aside into obscurity, Macromedia Dreamweaver and Microsoft FrontPage: it's the mid-2010s at last and we're all command-line jocks now.

Hacked-together for hacking fine sites together the (mostly imagined) "old-school way" — and probably more compelling (for users other than me, compared to other such tools) only in case you wish to command some awfully-intricate-templating powers and know to code up at least simple short pieces of Haskell (such as this 2-liner or this 5-liner) in a pinch: because I wrote it from the ground up with easy-speedy, "done-and-done" future extensibility chiefly in mind.

To be fair though: it does actually accomplish quite a lot without you ever needing to touch its codebase — alas you'll have to run-from-source (or compile from scratch) no less.

Quick start

  1. Fetch from Github
  2. Run with Haskell Stack (with MissingH already present)
    • first via: ~/haxtatic/src$ stack runghc ./main.hs* to see a note on how to really run this.
    • (* unless of course you prefer another way of building-running Haskell programs)
    • now run: ~/haxtatic/src$ stack runghc ./main.hs ~/where/ever/mydemosite.foo/
  3. Dig into the freshly generated and populated mydemosite.foo project-directory to grasp the bare essentials from the pre-created default single-page site in there.
  4. Dig into the project-directory (src-docs folder inside docs) for this very Docs site to figure out most-of-the-rest.
