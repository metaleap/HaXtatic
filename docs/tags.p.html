P-tags: per-page values (date, title, custom..)

{P{..}} tags include (built-in and custom) values specific to the current page.

Built-in

  • {P{Title}} → current page's first single-line <h1>'s entire inner markup
    — eg. for this page: P-tags: per-page values (date, title, custom..)
  • {P{Body}} → current page's entire (mostly already processed) markup source (only usable in theme.tmpl.html)
  • {P{FileName}} → the current page's output file name
    — eg. for this page: tags.p.html
  • {P{OrigPath}} → the current page's (relative local) source path
    — eg. for this page: ..\src-docs\pages\tags\2016.12.17.p.html
  • {P{BaseName}} → the current page's name minus file extension and section prefix:
    — eg. for this page named tags.p.html : p
  • {P{Date}} and {P{DateMyFormat}} → current 'page date'
    — eg. for this page: 2016/12/17
  • {P{PostText}} → current blok-post page's first single-line <p>'s entire inner text
    — eg. for this page: {P{..}} tags include (built-in and custom) values specific to the current page.

Custom

Define those like {P{%name:value}} in a page; then while that page is being processed, this value will (stripped of leading / trailing white-space) replace all encounters of {P{%name}} anywhere in the output (whether from page, theme.tmpl.html or blog.tmpl.html sources), including occurrences produced by T-tags or originating from other settings in haxtatic.config.

Any characters are acceptable for the name in between {P{% and  :, as well as the value in between  : and  }} .
