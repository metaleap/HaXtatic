X-tags: dynamic renderers

An {X| .. |} tag invokes a programmatic output renderer.

Such an "X-renderer" may be of a built-in type, or of a custom one developed in Haskell and linked into the haXtatic code-base manually. Naturally only the built-in types are covered here and in the X-renderers section.

Like T tags (and unlike P and B tags), there are no predefined X-tags, only those declared and set up in *.haxproj files — ie. before an X-tag can be embedded many times anywhere in the project, it is first to be defined once in a *.haxproj: to be given a name and some default settings.

Here are the built-in X-renderers ready to set up and try out:

  • hax/demoSimplest — "Hello World!"
  • hax/demoCfgArgs — "Hello, foo!"
  • hax/miniTag — to use eg. {X|bq: .. or {X|c: .. in place of verbosity such as <blockquote> .. </blockquote> or <code> .. </code>
  • hax/image — less-verbose images
  • hax/links — lean & neat navigation helper
  • hax/pageAnchors — gather and link to in-page contents
  • hax/snippet — elsewhere known as "custom controls", "components", "sub-templates", or "thinglets"..
  • hax/repeat — versatile content/data selector→rearranger→reformatter→renderer

demoSimplest

Outputs <h1>Hello World!</h1>, given:

  1. Declaration of an |X|hax/demoSimplest:myXtag X-tag in a *.haxproj file,
    • further configuring: (no settings)
  2. Usage anywhere: {X|myXtag |}

demoCfgArgs

Outputs <h1>Hello, greetname!</h1>, given:

  1. Declaration of an |X|hax/demoCfgArgs:myXtag or |X|hax/demoCfgArgs:myXtag:greetname X-tag in a *.haxproj file,
    • further configuring: (no other settings)
  2. Usage anywhere: {X|myXtag|} or {X|myXtag:greetname |}

miniTag

Outputs <somehtmltag ..>tag's inner-markup content</somehtmltag>, given:

  1. Declaration of an |X|hax/miniTag:myXtag:somehtmltag: .. X-tag in a *.haxproj file,
    • further configuring: htmlAtts
  2. Usage anywhere: {X|myXtag: tag's inner-markup content |}

image

Outputs <img .. /> or <a ..><img .. /></a>, given:

  1. Declaration of an |X|hax/image:myXtag:somereldirpath: .. X-tag in a *.haxproj file,
    • further configuring: lnkAtts, imgAtts
  2. Usage anywhere: {X|myXtag:pic/rel/path and optionally some description text |}

links

Outputs a sequence of <a ..> .. </a> or <outerhtmltag ..><a ..> .. </a></outerhtmltag> in a uniform manner, given:

  1. Declaration of an |X|hax/links:myXtag:outerhtmltag: .. X-tag in a *.haxproj file,
    • further configuring: htmlAtts, itemsFirst, itemsLast, wrapHref
  2. Usage anywhere: {X|myXtag: items = [..] , htmlAtts = [..] |}

pageAnchors

Outputs a sequence of <outerhtmltag ..><a ..> .. </a></outerhtmltag>, one per <gatherhtmltag> element found in the page's content source, given:

  1. Declaration of an |X|hax/pageAnchors:myXtag:gatherhtmltag: .. X-tag in a *.haxproj file,
    • further configuring: considerEmpty, htmlIfEmpty
  2. Usage anywhere: {X|myXtag:outerhtmltag: htmlAtts = [..] |}

snippet

Outputs the content of the named snippet with the specified variable substitutions.

  1. Declaration of an |X|hax/snippet:myXtag: .. X-tag in a *.haxproj file,
    • further configuring: vars, content
  2. Usage anywhere: {X|myXtag: vars = [..], content = ".." |}

repeat

Outputs a sequence of contents enumerated by the specified iterator.

  1. Declaration of an |X|hax/repeat:myXtag: .. X-tag in a *.haxproj file,
    • further configuring: prefix, suffix, joinwith, content
  2. Usage anywhere: {X|myXtag: over = .. , wrap = .. , order = .. , skip = .. , limit = .. |}
