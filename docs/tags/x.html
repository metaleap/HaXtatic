X-tags: dynamic renderers

An {X| .. |} tag invokes a programmatic output renderer (aka. X-renderer).

Such an "X-renderer" may be of a built-in type, or of a custom one developed in Haskell and linked into the HaXtatic code-base manually. Naturally only the built-in types are covered here and in the X-renderers section.

Like T-tags (and unlike P- and B-tags), there are no predefined X-tags, only those declared and set up in *.haxproj files — ie. before an X-tag can be embedded many times anywhere in the project, it is first to be defined once in a *.haxproj: to be given a name and some default settings.

Here are the built-in X-renderers ready to set up and try out:

  • hax.demoSimplest — "Hello World!"
  • hax.demoCfgArgs — "Hello, foo!"
  • hax.miniTag — to use eg. {X|bq: .. or {X|c: .. in place of verbosity such as <blockquote> .. </blockquote> or <code> .. </code>
  • hax.xmlEscape — HTML/XML-escape given content
  • hax.image — less-verbose images
  • hax.link — less-verbose links
  • hax.links — lean & neat navigation helper
  • hax.pageAnchors — gather and link to in-page contents
  • hax.snippet — elsewhere known as "custom controls", "components", "sub-templates", or "thinglets"..
  • hax.repeat — versatile content/data selector→rearranger→reformatter→renderer

demoSimplest

Outputs <h1>Hello World!</h1>.

  1. Declaration of an |X|hax.demoSimplest:myTag directive in a *.haxproj file,
    • further configuring: (no settings)
  2. Usage anywhere: {X|myTag |}

demoCfgArgs

Outputs <h1>Hello, greetname!</h1>.

  1. Declaration of an |X|hax.demoCfgArgs:myTag or |X|hax.demoCfgArgs:myXtag:greetname directive in a *.haxproj file,
    • further configuring: (no other settings)
  2. Usage anywhere: {X|myTag|} or {X|myXtag:greetname |}

miniTag

Outputs <somehtmltag ..>tag's inner-markup content</somehtmltag>.

  1. Declaration of an |X|hax.miniTag:myTag:somehtmltag: .. directive in a *.haxproj file,
    • further configuring: attr
  2. Usage anywhere: {X|myTag: tag's inner-markup content |}

image

Outputs <img .. /> or <a ..><img .. /></a>.

  1. Declaration of an |X|hax.image:myTag:some/rel/base/path: .. directive in a *.haxproj file,
    • further configuring: attrLink, attrImg
  2. Usage anywhere: {X|myTag:rel-to-base/path/no/whitespace and optionally some description text |}

xmlEscape

Outputs the given content with all occurrences of any of & < > " ' each appropriately entity-escaped for safe inclusion in markup.

  1. Declaration of an |X|hax.xmlEscape:myTag directive in a *.haxproj file,
    • further configuring: (no settings)
  2. Usage anywhere: {X|myTag: any "content' to <> XML&HTML-escape |}

link

Outputs <a ..>..</a>.

  1. Declaration of an |X|hax.link:myTag:some/rel/base/path: .. directive in a *.haxproj file,
    • further configuring: attr , xmlEscape , uriAutoExt
  2. Usage anywhere: {X|myTag:rel-to-base/path/no/whitespace and then the link text text |}

links

Outputs a sequence of <a ..> .. </a> or <outerhtmltag ..><a ..> .. </a></outerhtmltag> in a uniform manner.

  1. Declaration of an |X|hax.links:myTag:outerhtmltag: .. directive in a *.haxproj file,
    • further configuring: attr, itemsFirst, itemsLast, wrapHref
  2. Usage anywhere: {X|myTag: items = [..] , attr = [..] |}

pageAnchors

Outputs a sequence of <outerhtmltag ..><a ..> .. </a></outerhtmltag>, one per <gatherhtmltag> element found in the page's content source.

  1. Declaration of an |X|hax.pageAnchors:myTag:gatherhtmltag: .. directive in a *.haxproj file,
    • further configuring: considerEmpty, outputIfEmpty, xmlEscapeHref
  2. Usage anywhere: {X|myTag:outerhtmltag: attr = [..] |}

snippet

Outputs the content of the named snippet with the specified variable substitutions.

  1. Declaration of an |X|hax.snippet:myTag: .. directive in a *.haxproj file,
    • further configuring: vars, flags, content
  2. Usage anywhere: {X|myTag: vars = [..], flags = [..], content = ".." |}

repeat

Outputs a sequence of contents enumerated by the specified iterator.

  1. Declaration of an |X|hax.repeat:myTag: .. directive in a *.haxproj file,
    • further configuring: prefix, suffix, joinVia, content
  2. Usage anywhere: {X|myTag: over = .. , wrapEach = .. , order = .. , skip = .. , limit = .. , more = .. |}
