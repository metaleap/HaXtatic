T-tags: named plain text/markup snippets

{T{..}} tags directly include named text values that you defined once in your {T{HaxConfLink}} and then reuse wherever needed.

{T{Briefly}}

  • Single-line definitions in {T{HaxConfLink}}:
    T:AuthBlurb: The author is a serial amateur dabbling in role-playing on XingedIn.
  • Usage anywhere: {T{AuthBlurb}}

Definition in {T{HaxConfi}}

A single line beginning with with T:yourtextname: followed by any text value in {T{HaxConfLink}} defines a named text.

Example: this very site in its {T{HaxConfLink}} contains this line:
T:SiteTitle: {T{SiteTitle}}
(the white-space isn't strictly required) — indeed you'll find this named text value included in the browser/tab title of every page within this site.

Names can contain any characters except colons.

Values can contain any characters but will be stripped of all leading and trailing white-space.

Embed in any markup

Just use {T{yourtextname}} anywhere you author your content to include your named text value right in-place during site generation — content source pages, templates, posts, even your {T{HaxConfLink}} itself will work.

Undefined text-value names will not be replaced and so remain in the output.

Here's how the <body> of this site's {T{ThemeHtmlLink}} (and thus that of every single output page) begins:
<h4>{\T{SiteTitle}}</h4>

See also

  • A more versatile equivalent of this functionality is offered by the {T{HaxXSnippets}} X-tag, which let's you pass variable substitutions with all your named-text inclusions.
  • To define named text values on a per-page basis (reuse within a single page but unknown outside it), use custom {T{HaxPTag}}s.
