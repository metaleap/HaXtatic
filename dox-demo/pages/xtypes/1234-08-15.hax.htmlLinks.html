  • {X|c:attr = [("name","value") , ..]|} — the output outer-tag's markup attributes as name-value pairs
    • Attribute names starting with {X|c: &|} ampersand are only output (without it, of course) for path-wise "current/selected/active" links with respect to the current page-context (see below paragraph for the approximate logic used here)
    • use of any such dynamic {X|c: &|} attribute names switches this X-tag's {X|lt:index#stages stage|} from Early (the default) into Page.
  • {X|c:itemsFirst = [("uri","text") , ..]|} — the links to always render first
  • {X|c:itemsLast = [("uri","text") , ..]|} — the links to always render last
  • {X|c:wrapHref = ("","")|} — the prefix-to-prepend & suffix-to-append to every rendered {X|c:<a>|} element's {X|c:href|} attribute value
  • {T|Xnoesc|}
    • |} {T|Xargs|}

    For example, {X|mysidenav: items=[("foo","Foo"),("bar","Bar")] , attr=[("class","my-foo")] |} to output:

    <li class="my-foo"><a href="foo">Foo</a></li>
<li class="my-foo"><a href="bar">Bar</a></li>

    For example, {X|mytopnav|} to render in all generated {X|c:/blog/*|} output files:

    <div class="my-nav"><a href="/index.html">Home</a></div>
<div class="my-nav my-nav-sel"><a href="/blog/index.html">My Blog</a></div>

    — whereas all others would see the same, minus the {X|c:my-nav-sel|} appendum.
