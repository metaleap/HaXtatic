{X|dox-xthead|} {T|Xearly|} {T|Xpurpose|}

In a similar spirit to {X|lx:hax.miniTag hax.miniTag|} and {X|lx:hax.htmlImage hax.htmlImage|}, this one is for saving precious keystrokes with {X|c:<a ..>|}-rich HTML contents requiring repetetive use of quickly-becoming-too-verbose markup.

{T|Xcfg|} {T|Xdecl|} 
|X|hax.htmlLink:lb: {P|/basics|}

|X|hax.htmlLink:lt: /tags:
	attr = [ ("class",
	"foo-sitelink")],
	xmlEscape =
	(True , False),
	uriAutoExt = ".html"

|X|hax.htmlLink: lx:
	../xtypes/
{P|X|dox-xtdecl: vars= [ ("xn" , "hax.htmlLink"), ("xf" , "The relative base path for all links from this X-tag (here {X|c:../xtypes|}),") ], flags = [], content=>
  • {X|c:attr = [ .. ]|} — a List of all the tag attributes (each denoted in a {X|c:("name","value")|} pair) to always append (in addition to automatic {X|c:href|}) to the {X|c:<a>|} output by all instances of this X-tag
  • {X|c:xmlEscape = (.. , ..)|} — a tuple of 2 toggles (the first for the {X|c:href|} and the second for the inner content of the {X|c:<a>|} output by all instances of this X-tag) indicating whether to auto-HTML/XML-escape ({X|c:True|}) or not ({X|c:False|})
  • {X|c:uriAutoExt = ".."|} — either {X|c: ""|} (empty) or a file name extension to always append (if missing) to the part of {X|c:href|} prior to {X|c: #|} of the {X|c:<a>|} output by all instances of this X-tag
    • |} {T|Xargs|}

    Usage for the 2nd example from above: {X|c: {X|lt:x#Delaying%20X-tags Delaying X-tags |} |} would output {X|c: <a class="foo-sitelink" href="/tags/x.html#Delaying%20X-tags">Delaying X-tags</a> |} — it follows that:

    • the (X-tag-)relative URI path comes first, followed by
    • a white-space and the link element's inner content;
    • any white-space in the URI path is to be replaced, as is standard, with {X|c:%20|};
    • if {X|c:xmlEscape|} is {X|c:False|} for {X|c:href|}, take care that the combined URI will be HTML-escaped
