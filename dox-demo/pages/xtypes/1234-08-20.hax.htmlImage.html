{X|dox-xthead|} {T|Xearly|} {T|Xpurpose|}

In a similar spirit to {X|lx:hax.miniTag hax.miniTag|} and {X|lx:hax.htmlLink hax.htmlLink|}, this one is for saving precious keystrokes with {X|c:<img ..>|}-rich HTML contents requiring repetetive use of quickly-becoming-too-verbose markup.

{T|Xcfg|} {T|Xdecl|} 
|X|hax.htmlImage:logo:
	../pics/logos

|X|hax.htmlImage: photo : {P|/pics/photos/|}:
	attrLink = [ ("class","foo-photo") ],
	attrImg = [("class" , "foo-roundborder")
		, ("onload","myGallerize(this)")
		],
	xmlEscape = True
{P|X|dox-xtdecl: vars= [ ("xn" , "hax.htmlImage"), ("xf" , "The relative base path for all images from this X-tag (here {X|``lt:p#relpaths ensuring root-relative |}: {X|c: {P| /pics/photos |} |}),") ], flags = [], content=>
  • {X|c:attrLink = [ .. ]|} — a List of all the tag attributes (each denoted in a {X|c:("name","value")|} pair) to always append (in addition to automatic {X|c:href|} and {X|c:title|}) to the outer {X|c:<a>|} element
    • if empty (the default), no such outer {X|c:<a>|} element will be rendered (only the {X|c:<img>|} element itself)
  • {X|c:attrImg = [ .. ]|} — a List of all the tag attributes (each denoted in a {X|c:("name","value")|} pair) to always append (in addition to automatic {X|c:src|} and {X|c:alt|}) to the inner {X|c:<img>|} output by all instances of this X-tag
  • {X|c:xmlEscape = ..|} — either {X|c:False|} (the default) or {X|c:True|}; whether to auto-HTML/XML-escape the automatic {X|c:title|} / {X|c:alt|} attribute values
    • |} {T|Xargs|}

    Usage for the above example: {X|c: {X|photo:eire%202016/IMG_5.jpg Look, a 'geysir'! |} |} would output, depending on the current-page URL, perhaps something like: {X|c: <a class="foo-photo" href="../pics/photos/eire%202016/IMG_5.jpg" title="Look, a &apos;geysir&apos;!"> <img class="foo-roundborder" onload="myGallerize(this)" src="../pics/photos/eire%202016/IMG_5.jpg" alt="Look, a &apos;geysir&apos;!" /></a> |} — verbose! It follows that:

    • the (X-tag-)relative image path comes first, followed optionally
    • by a white-space and any manner of picture description, if any;
    • any white-space in the picture URI path is to be replaced, as is standard, with {X|c:%20|};
    • if {X|c:xmlEscape|} is {X|c:False|}, the picture description has to be provided HTML-escaped: manually or wrapped inside an {X|lx:hax.xmlEscape hax.xmlEscape|}
