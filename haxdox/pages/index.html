Overview

These are the user docs on HaXtatic:

  • neither the first, nor the last .. neither the most powerful, nor the most minimalist .. simply just, wait for it ..
  • yet another static-site generator! You heard that right.
  • So step aside into obscurity, Macromedia Dreamweaver and Microsoft FrontPage! It's the mid-2010s at last and we're all command-line jocks now.

Hacked-together for hacking fine things together the (broadly imagined) "old-school way" — and probably really only possibly compelling (for users other than me, compared to other such tools) in case you wish to command some awfully-intricate-templating powers and know to code up at least simple pieces of Haskell in a pinch: because I have written it from the ground up with easy-speedy, "done-and-done" future extensibility chiefly in mind.

To be fair though: it does actually accomplish quite a lot without ever needing to touch its codebase — alas you'll have to run-from-source (or compile from scratch) no less.

Quick start

  1. Fetch from Github
  2. Run with Haskell Stack (with MissingH installed)
    • first via: stack runghc ./main.hs to see a note on how to really run this.
    • (unless of course you prefer to another way of building-running Haskell programs)
    • now run: stack runghc ./main.hs ~/where/ever/mydemosite.foo/
  3. Dig into the freshly generated and populated mydemosite.foo project-directory to grasp the bare essentials from the pre-created default single-page site in there.
  4. Dig into the project-directory (haxdox folder inside docs) for this very Docs site to figure out most-of-the-rest.
