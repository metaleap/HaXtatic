Overview

These are the user docs on HaXtatic:

  • neither the first, nor the last
  • neither the most powerful, nor the most minimalist
  • simply just yet-another static-site generator.
  • You heard that right.
  • So forget about Dreamweaver and FrontPage, we're all command-line jocks now. It's the 2010s after all!

Hacked-together for hacking fine sites together the (perceived) old-school way — and probably really only possibly compelling (for users other than me, and compared to other such programs) in case you know to code up at least simple pieces of Haskell: because I have written it from the beginning with easy-speedy, "done-and-done" future extensibility chiefly in mind.

Quick start:

  1. Fetch from Github
  2. Run with Haskell Stack (with MissingH installed)
    • first via: stack runghc ./main.hs to see a note on how to really run this.
    • (unless of course you prefer to another way of building-running Haskell programs)
    • now run: stack runghc ./main.hs ~/where/ever/mydemosite.foo/
  3. Dig into the freshly generated and populated mydemosite.foo project-directory to grasp the bare essentials from the pre-created default single-page site in there.
  4. Dig into the project-directory (haxdox folder inside docs) for this very Docs site to figure out most-of-the-rest.
  5. All the nitty-gritty details follow:
