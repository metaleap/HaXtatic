Project directories

Any project resides in its own project-directory, with a predefined (auto-generated) minimum-structure of 4 specific files plus these 4 folders:

  • build (generated output)
  • pages (all your authored content pages as likely-mostly-unstyled/unthemed/inner-body-only HTML files)
  • static (raw files to copy unprocessed into build)
  • posts (not important for now)

Get started

Run HaXtatic passing "the project-directory to process" and any missing parts from the above will be (re)created in-place.

Example, say we want to create a new site project named mydemosite.foo (inside our existing ~/dev/ in this case). Just run stack runghc main.hs ~/dev/mydemosite.foo (or your-own-compiled-binary of course) and we get:

Now you just create/modify contents/templates/settings/etc. within this structure and then simply run the same command-line again specifying the very same directory path to fill build with your freshly re-generated static site.

4 standard folders

Directory build:

Contains the complete generated site after a successful run. No clean-ups done here by HaXtatic in-between runs at this point.

Directory static:

All contents (if any) will just be copied over straight into build (with no processing) as the first step.

Directory posts:

Leave it empty for now, described later.

Directory pages:

All files in here will then be interpreted as "likely mostly untemplated / unthemed / unstyled, essentially purely-content" HTML pages to be templated (resulting output page goes into build); if this has no files at all, a default index.html will be created to get you started with.

No sub-folders for now: their files won't just carry over into build quite in the same fashion (and their presence is reserved for a more advanced feature explained later) — so only top-level pages for your simple starter site contents for now.

4 standard files

Always created if missing:

  • haxtatic.config — initially near-empty file (and can well be empty) that you later stuff with all sorts of custom project-specific definitions used in your content source pages or from the following default templates:
  • theme.tmpl.html — contains the complete outer HTML scaffolding markup that every generated page will embed its own unique contents into. If empty, all generated output pages will be empty. Needs at least the {P{Body}} tag for output pages to contain their source pages' contents.
  • blog.tmpl.html — unused in the initial minimum default structure and can be empty; discussed later.

Only created in new projects:

  • pages/index.html — only created whenever the pages directory is totally devoid of any files, as a handy example content source page to get you started with.
